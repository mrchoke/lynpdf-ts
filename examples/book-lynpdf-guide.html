บทที่ 1: แนะนำ LynPDF

1.1 LynPDF คืออะไร?

LynPDF เป็นไลบรารี TypeScript สำหรับแปลง HTML/CSS เป็นไฟล์ PDF คุณภาพสูง โดยไม่ต้องพึ่งพาเบราว์เซอร์ (Headless Browser) เช่น Chrome หรือ Puppeteer ทำให้มีน้ำหนักเบา ทำงานรวดเร็ว และเหมาะสำหรับใช้งานฝั่ง Server-side

LynPDF ออกแบบมาเพื่อรองรับภาษาไทยเป็นพิเศษ ด้วยการตัดคำภาษาไทยอัตโนมัติ (Thai word segmentation) ผ่าน Intl.Segmenter รองรับฟอนต์ไทยหลายตระกูล และจัดการสระลอย/วรรณยุกต์ได้ถูกต้อง

1.2 คุณสมบัติเด่น

✓ Yoga Layout Engine

ใช้ Yoga (จาก Meta/Facebook) เป็น Flexbox layout engine ทำให้รองรับ CSS Flexbox properties ได้ครบถ้วน เช่น flex-direction, flex-wrap, justify-content, align-items

✓ PDFKit Rendering

ใช้ PDFKit สำหรับสร้างไฟล์ PDF รองรับ compression, metadata, bookmarks และ hyperlinks

✓ ฟอนต์ไทยครบทุกตระกูล

รองรับฟอนต์ Sarabun, Prompt, Kanit, Mitr, Chakra Petch, และ TLWG fonts ทั้งหมด 13 ตระกูล

✓ SVG Support

แสดงผล SVG แบบ Vector ภายใน PDF ผ่าน svg-to-pdfkit รองรับ gradient, path, shape ต่างๆ

✓ Color Emoji

รองรับ Color Emoji ด้วย Twemoji PNG fallback แสดงอิโมจิสีสันสดใสใน PDF

บทที่ 2: การติดตั้งและเริ่มต้นใช้งาน

2.1 ความต้องการของระบบ

รายการ เวอร์ชันขั้นต่ำ หมายเหตุ

2.2 การติดตั้ง

bun add lynpdf

หรือติดตั้งจาก source:

git clone https://github.com/example/lynpdf.git

cd lynpdf

bun install

2.3 ตัวอย่างการใช้งานพื้นฐาน

import { PDFCreator } from 'lynpdf';

const creator = new PDFCreator();

const html = '<h1>สวัสดี LynPDF</h1>';

const css = 'h1 { color: #e94560; }';

await creator.createPDF(html, css, 'output.pdf');

Tip: LynPDF รองรับทั้ง inline styles (ใน HTML attribute) และ external CSS (ผ่าน parameter) สามารถเลือกใช้ได้ตามความเหมาะสม

2.4 การใช้งานผ่าน CLI

bunx lynpdf input.html -o output.pdf

bunx lynpdf input.html -c styles.css -o output.pdf

bunx lynpdf input.html --no-compress -o output.pdf

ตัวเลือก CLI ที่สำคัญ:

ตัวเลือก คำอธิบาย ค่าเริ่มต้น
-o, --output ชื่อไฟล์ PDF ที่จะสร้าง output.pdf
-c, --css ไฟล์ CSS ภายนอก —
--no-compress ปิด PDF compression compress: true
-v, --verbose แสดงรายละเอียดการทำงาน false
บทที่ 3: สถาปัตยกรรมระบบ

3.1 Rendering Pipeline

LynPDF ทำงานตามลำดับขั้นตอน (Pipeline) ดังนี้:

1. HTML Parsing (parse5)
↓
2. CSS Parsing (css-tree)
↓
3. Style Resolution (Selector matching)
↓
4. Layout Calculation (Yoga Flexbox)
↓
5. Page Break (page-break-inside: avoid)
↓
6. PDF Rendering (PDFKit)

3.2 โครงสร้างโปรเจกต์

โฟลเดอร์/ไฟล์ หน้าที่
src/pdf-creator.ts API หลัก — ประสานงานทุกขั้นตอน
src/layout/layout-engine.ts แปลง DOM → Yoga Layout → LayoutNode tree
src/layout/style-resolver.ts จับคู่ CSS selector กับ DOM element
src/layout/yoga-mapper.ts แปลง CSS properties → Yoga node properties
src/renderer/pdf-renderer.ts วาด LayoutNode tree ลง PDFKit document
src/renderer/color-parser.ts แปลงสี CSS (hex, rgb, rgba, named) → PDFKit color
src/text/text-measurer.ts วัดความกว้างข้อความ, นับบรรทัด, ตัดคำไทย
src/text/emoji-handler.ts ดาวน์โหลดและแสดงผล Color Emoji (Twemoji)

3.3 การจัดการหน้า (Pagination)

LynPDF ใช้วิธี "virtual scroll" — Yoga คำนวณ layout เป็นเอกสารเดียวยาวต่อเนื่อง จากนั้น renderer จะแยกเนื้อหาออกเป็นหน้าๆ โดยใช้สูตร:

pageIndex = Math.floor(node.y / PAGE_HEIGHT)

localY = node.y - pageIndex * PAGE_HEIGHT

ขนาดหน้า A4 = 595.28 x 841.89 points, ระยะขอบ = 50 points ทุกด้าน

หมายเหตุ: page-break-before/after ทำงานในขั้นตอน Layout (ก่อน rendering) โดยเลื่อน node ลงมายังจุดเริ่มต้นหน้าถัดไป ทำให้ Yoga สามารถปรับตำแหน่ง descendant ทั้งหมดให้ถูกต้อง

5.2 การตัดคำภาษาไทย

LynPDF ใช้ Intl.Segmenter (Thai locale) สำหรับตัดคำภาษาไทยอัตโนมัติ ช่วยให้ข้อความยาวสามารถขึ้นบรรทัดใหม่ได้อย่างถูกต้องตามหลักภาษาไทย

ตัวอย่างข้อความยาว: สำนักงานพัฒนาธุรกรรมทางอิเล็กทรอนิกส์ หรือ สพธอ. เป็นหน่วยงานภายใต้กระทรวงดิจิทัลเพื่อเศรษฐกิจและสังคม มีหน้าที่ส่งเสริมและสนับสนุนการทำธุรกรรมทางอิเล็กทรอนิกส์ของประเทศไทย รวมถึงการพัฒนามาตรฐานและกฎเกณฑ์ที่เกี่ยวข้อง เพื่อสร้างความเชื่อมั่นและความปลอดภัยในการทำธุรกรรมออนไลน์

หมายเหตุ: การตัดคำใช้ newline character (\n) แทน zero-width space (U+200B) เพื่อป้องกันปัญหา glyph ที่หายไปในบางฟอนต์

5.3 Font Fallback

LynPDF มีระบบ Font Fallback อัตโนมัติ — เมื่อฟอนต์หลักไม่มี glyph สำหรับอักขระบางตัว (เช่น ✓ ✗ ★) ระบบจะสลับไปใช้ฟอนต์สำรองโดยอัตโนมัติ:

ตัวอย่าง: ✓ สำเร็จ ✗ ล้มเหลว ★ สำคัญ → ถัดไป ← ย้อนกลับ
