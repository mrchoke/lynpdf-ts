LynPDF Creator 🐱

HTML/CSS & Markdown → PDF converter with Thai language support, color emoji, flexbox layout, and OpenType shaping — no headless browser required.

⭐ View on GitHub 📦 npm package
$ bun add lynpdf click to copy

Features

Everything you need to produce professional PDFs from HTML, CSS, and Markdown.

📄

HTML + CSS → PDF

Convert standard HTML/CSS to pixel-perfect PDFs without any headless browser. Pure JavaScript/WASM pipeline.

🇹🇭

Thai Language

20+ Thai font families (Google Fonts + TLWG), word segmentation via Intl.Segmenter, stacked vowels/tone marks, and text-align: justify.

🕉️

Pali / Sanskrit

Correct rendering of นิคหิต ( ํ ), พินทุ ( ฺ ), ทัณฑฆาต ( ์ ), stacked diacritics via HarfBuzz GSUB/GPOS.

😀

Color Emoji

Full-color Twemoji PNG rendering with auto-download and local caching.

📐

Flexbox Layout

Powered by Meta's Yoga Layout engine (C++ compiled to WASM) for accurate CSS Flexbox positioning.

🔡

OpenType Shaping

HarfBuzz GSUB (ligatures: fi fl ff) + GPOS (kerning pairs) for all fonts via WebAssembly.

🔤

Font Support

TTF, OTF (IBM Plex Sans), and Variable Fonts (Inter Variable) with automatic weight/style resolution.

📊

Tables

Full HTML tables with <thead> repeat on every page; disable with -lynpdf-repeat: none.

🖼️

SVG & Images

Inline SVG vector rendering, <img> tags, and CSS backgrounds.

🖨️

Page Rules

@page margins, named sizes (A0–A6, letter, legal), landscape/portrait, counter(page) headers & footers.

📝

Markdown → PDF

Convert .md files directly; syntax highlighting (190+ languages), containers, GitHub Alerts, color boxes, Mermaid diagrams.

⚡

Fast

Bun runtime, font metric caching, Twemoji PNG caching (~500–700 ms for complex documents).

Quick Start

Get up and running in under a minute.

Installation

Bun (recommended) 
# Install Bun
curl -fsSL https://bun.sh/install | bash

# Add LynPDF to your project
bun add lynpdf
npm / pnpm / yarn 
# npm
npm install lynpdf

# pnpm
pnpm add lynpdf

# yarn
yarn add lynpdf

CLI Usage

Terminal 
# Basic: HTML → PDF
lynpdf report.html

# With external CSS
lynpdf report.html -c styles.css -o output.pdf

# From stdin
echo "<h1>Hello PDF</h1>" | lynpdf --stdin -o hello.pdf

# Markdown → PDF (auto-detected by .md extension)
lynpdf README.md -o readme.pdf

# With page size and margin options
lynpdf invoice.html -c theme.css -o invoice.pdf --page-size A4 --margin 50

API Usage

TypeScript — Basic 
import { PDFCreator } from 'lynpdf'

const creator = new PDFCreator()

// From strings
await creator.createPDF(
  '<h1>Hello World</h1><p>สวัสดีชาวโลก 🌍</p>',
  'h1 { color: #1a1a2e; font-size: 28px; }',
  'output.pdf'
)

// As buffer (for APIs / HTTP responses)
const buffer = await creator.createPDFBuffer(html, css)
TypeScript — Markdown 
import { PDFCreator } from 'lynpdf'

const creator = new PDFCreator()

// From Markdown string
await creator.createPDFFromMarkdown(
  '# Hello\n\n```ts\nconst x = 42\n```',
  '',
  'output.pdf'
)

// From .md file
await creator.createPDFFromMarkdownFile(
  'README.md', '', 'readme.pdf'
)

Framework Integration

Express.js 
import { PDFCreator } from 'lynpdf'
import express from 'express'

const app = express()
const creator = new PDFCreator()

app.get('/invoice/:id', async (req, res) => {
  const html = renderInvoiceTemplate(req.params.id)
  const buffer = await creator.createPDFBuffer(html, css)
  res.setHeader('Content-Type', 'application/pdf')
  res.send(buffer)
})
Next.js API Route 
import { PDFCreator } from 'lynpdf'

export async function GET(request: Request) {
  const creator = new PDFCreator()
  const buffer = await creator.createPDFBuffer(
    '<h1>Report</h1>',
    'h1 { color: blue; }'
  )
  return new Response(buffer, {
    headers: { 'Content-Type': 'application/pdf' }
  })
}

Architecture

How LynPDF transforms HTML/CSS and Markdown into professional PDFs.

HTML → parse5 → DOM tree CSS → css-tree → Stylesheet AST ↓ StyleResolver (CSS selector cascade) ↓ LayoutEngine (Yoga Flexbox — WASM) ↓ Post-Layout Passes: • applyPageFlow (page-breaks, footer-zone, thead integrity) • applyOrphansWidows • applyTheadRepeatShift (-lynpdf-repeat) • anchorPageBreaks ↓ PDFRenderer (PDFKit + HarfBuzz + Twemoji) ↓ PDF file

Key Technologies

Yoga Layout

Meta's cross-platform Flexbox engine (C++ → WASM). No native compile step needed.

PDFKit

PDF generation library for Node.js — pure JavaScript.

HarfBuzz (WASM)

OpenType shaping: Thai floating vowels, ligatures (fi/fl), kerning — no system libharfbuzz needed.

fontkit

TTF/OTF/Variable font parser with glyph metrics — pure JavaScript.

parse5

Spec-compliant HTML5 parser — pure JavaScript.

Twemoji

Twitter's open-source color emoji — PNGs fetched from CDN and cached locally.

Requirements

Minimal system dependencies — all heavy lifting is done via WebAssembly.

Requirement Version Notes
Bun ≥ 1.0 Recommended runtime — fast, native ESM, built-in TypeScript
Node.js ≥ 18 Alternative runtime; required for Intl.Segmenter (th-TH)
TypeScript ≥ 5.0 Peer dependency; needed only for type declarations
WebAssembly note: harfbuzzjs (OpenType shaping) and yoga-layout (Flexbox engine) ship as self-contained WASM binaries. No system-level libharfbuzz or libyoga needed. Bun ≥ 1.0 and Node.js ≥ 18 already include WASM support.
