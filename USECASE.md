การสร้าง Test Suite สำหรับ PDF Renderer เป็นขั้นตอนที่สำคัญมากครับ เพราะการแปลง HTML/CSS เป็นหน้ากระดาษ (Paged Media) มี Edge Cases เยอะมาก (นี่คือจุดที่ทำให้ไลบรารีหลายตัว "หน้าแตก" เมื่อเจอผู้ใช้งานจริง)

เพื่อความครอบคลุมและเป็นระบบ ผมจัดกลุ่มรายการ HTML + CSS Test Cases ที่คุณต้องสร้างไว้เป็น 6 หมวดหมู่หลัก (พร้อมจุดประสงค์การเทสต์) ดังนี้ครับ:

---

### 1. หมวดภาษาไทยและ Typography (Complex Text Layout - จุดปราบเซียน)

หมวดนี้คือตัวชี้วัดว่าไลบรารีของคุณพร้อมสำหรับผู้ใช้งานชาวไทยหรือไม่

* **Test 1.1: สระลอยและวรรณยุกต์ซ้อน (Glyph Positioning):** * **HTML:** พิมพ์คำที่มีการซ้อนระดับอักษร เช่น `น้ำ`, `ผู้`, `ป่า`, `ที่`, `ฎีกา`, `ญัตติ`, `ปู่`, `ตุ๊ย`
* **Expect:** สระบน-ล่างต้องไม่ทับซ้อนกัน ไม่ลอยห่างเกินไป และตำแหน่งต้องเป๊ะตามฟอนต์


* **Test 1.2: การตัดคำภาษาไทย (Thai Line Breaking):**
* **HTML:** ใส่ Text พารากราฟยาวๆ ที่ไม่มีเว้นวรรค
* **Expect:** โปรแกรมต้องขึ้นบรรทัดใหม่เป็นคำๆ ไม่ตัดกลางคำ (เช่น `ประเท-ศไทย` แบบนี้คือหน้าแตก)


* **Test 1.3: Text Alignment (Justify):**
* **CSS:** `text-align: justify;`
* **Expect:** การกระจายช่องไฟภาษาไทยต้องไม่ทำให้สระ/วรรณยุกต์หลุดออกจากพยัญชนะต้น


* **Test 1.4: Font Fallback & Multiple Scripts:**
* **HTML:** `สวัสดี Hello こんにちは`
* **Expect:** ถ้าระบุฟอนต์หลักที่ไม่มีภาษาญี่ปุ่น ระบบต้องดึงฟอนต์สำรอง (Fallback) มาแสดงผลอักษรญี่ปุ่นได้โดยไม่ขึ้นเป็นสี่เหลี่ยม (Tofu)



### 2. หมวด Paged Media & Pagination (การจัดการหน้ากระดาษ)

นี่คือสิ่งที่แยก Web Browser ออกจาก PDF Creator อย่าง WeasyPrint

* **Test 2.1: Page Size & Margins:**
* **CSS:** `@page { size: A4 portrait; margin: 2cm; }` และทดสอบไซส์ประหลาดๆ
* **Expect:** ขนาดไฟล์ PDF ที่ออกมาต้องตรงกับมิลลิเมตร/นิ้วที่กำหนดเป๊ะ


* **Test 2.2: Page Breaks (การบังคับขึ้นหน้าใหม่):**
* **CSS:** `page-break-before: always;`, `page-break-after: always;`
* **Expect:** เนื้อหาต้องถูกดีดไปหน้าถัดไปอย่างถูกต้อง


* **Test 2.3: ป้องกันการถูกหั่นครึ่ง (Page Break Inside):**
* **CSS:** `page-break-inside: avoid;` คลุม `<div class="card">` หรือ `<img>` ที่อยู่คาบเกี่ยวระหว่างหน้า 1 และ 2
* **Expect:** ออบเจกต์นั้นต้องถูกดันลงไปอยู่หน้า 2 ทั้งก้อน ไม่ถูกหั่นครึ่ง


* **Test 2.4: Orphans & Widows (บรรทัดหลงฝูง):**
* **CSS:** `orphans: 3; widows: 3;` กับพารากราฟยาวๆ ข้ามหน้า
* **Expect:** ไม่ควรมีบรรทัดของพารากราฟหลุดไปอยู่เดี่ยวๆ ที่ด้านบนหรือด้านล่างสุดของหน้ากระดาษเพียงบรรทัดเดียว



### 3. หมวด Tables (โครงสร้างข้อมูล)

ตารางคือสิ่งที่ผู้ใช้ชอบเอามาทำรายงาน และเป็นสิ่งที่เรนเดอร์ยากที่สุดเมื่อต้องข้ามหน้า

* **Test 3.1: ตารางยาวข้ามหน้า (Long Tables):**
* **HTML:** `<table>` ที่มี `<tr>` สัก 100 แถว
* **Expect:** ตารางต้องไหลลงหน้าถัดไปได้โดยโครงสร้างไม่พัง


* **Test 3.2: Repeating Headers / Footers:**
* **HTML:** ใช้ `<thead>` และ `<tfoot>` ในตารางที่ยาวข้ามหน้า
* **Expect:** หัวตารางและท้ายตารางต้องถูกพิมพ์ซ้ำ (Repeat) อัตโนมัติในทุกหน้าที่ตารางปรากฏ


* **Test 3.3: Rowspan & Colspan ผสม Page Break:**
* **HTML:** เซลล์ตารางที่ทำ `rowspan="5"` แต่ดันถูกขึ้นหน้าใหม่ตรงแถวที่ 3 พอดี
* **Expect:** (จุดนี้ไลบรารีส่วนใหญ่พัง) เลย์เอาต์ตารางหน้าถัดไปต้องไม่เบี้ยว


* **Test 3.4: Border Collapse:**
* **CSS:** `border-collapse: collapse;`
* **Expect:** เส้นขอบตารางต้องทับซ้อนกันพอดี ไม่หนาเป็น 2 เท่า



### 4. หมวด Box Model & Layout

การวาดกล่องข้อความให้เหมือนที่ตาเห็นบนเบราว์เซอร์

* **Test 4.1: Flexbox เบื้องต้น:**
* **CSS:** `display: flex; flex-direction: row; justify-content: space-between;`
* **Expect:** เลย์เอาต์ต้องจัดเรียงถูกต้อง (หมายเหตุ: WeasyPrint เพิ่งรองรับ Flexbox ได้สมบูรณ์เมื่อไม่นานมานี้ นี่คือจุดขายของ TS ถ้าทำได้ดี)


* **Test 4.2: Absolute / Fixed Positioning:**
* **CSS:** `position: absolute; top: 0; right: 0;` (เทียบกับหน้ากระดาษ)
* **Expect:** ตำแหน่งต้องอ้างอิงจากมุมของกระดาษ (Page Box) ไม่ใช่มุมของหน้าจอ


* **Test 4.3: Overflow & Clipping:**
* **CSS:** `overflow: hidden;` กับกล่องที่มีเนื้อหาล้น
* **Expect:** เนื้อหาที่ล้นต้องถูกตัดทิ้ง ไม่โผล่ออกมาใน PDF



### 5. หมวด Graphics, Colors & Images

ทดสอบความสามารถในการวาดและจัดการสี (เชื่อมโยงกับเรื่อง PDF 1.4 Transparency ที่คุยกันไปก่อนหน้า)

* **Test 5.1: RGBA & Opacity (Transparency):**
* **CSS:** กล่อง 2 ใบซ้อนทับกัน ใบหน้าใช้ `background-color: rgba(255, 0, 0, 0.5);`
* **Expect:** สีต้องผสมกันตามหลักการของ Alpha Channel (ไม่กลายเป็นสีทึบ)


* **Test 5.2: SVGs & Raster Images:**
* **HTML:** แทรก `<img src="vector.svg">` และ `<img src="high-res.jpg">`
* **Expect:** SVG ต้องยังคงเป็น Vector ใน PDF (ซูมแล้วไม่แตก) และรูป JPG ต้องถูก Scale ให้พอดีกับกรอบโดยสัดส่วนไม่เพี้ยน (aspect-ratio)


* **Test 5.3: CSS Borders & Border-Radius:**
* **CSS:** `border-radius: 50%; border: 5px dashed black;`
* **Expect:** มุมต้องโค้งมนจริงและเส้นประต้องเรนเดอร์ถูกต้อง



### 6. หมวด Interactive & Document Metadata

ทดสอบฟีเจอร์ระดับ PDF ที่มากกว่าแค่แผ่นกระดาษ

* **Test 6.1: Internal Links & Bookmarks:**
* **HTML:** มี `<a href="#chapter1">` และปลายทางคือ `<h1 id="chapter1">`
* **Expect:** คลิกใน PDF แล้วต้องกระโดดไปหน้าที่ถูกต้อง และควรสร้างเป็น Table of Contents (Outline) ด้านข้างของ PDF Viewer ได้


* **Test 6.2: External Links:**
* **HTML:** `<a href="https://google.com">`
* **Expect:** คลิกแล้วเปิดเบราว์เซอร์ออกไปหน้าเว็บได้
